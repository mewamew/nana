## 实现要求

### 1. `frontend/src/hooks/useLipSync.js`

```javascript
import { useRef, useCallback } from 'react'

// 音量 → 嘴巴张合映射参数
const VOLUME_THRESHOLD = 0.02   // 低于此音量视为静音（闭嘴）
const VOLUME_SCALE = 2.5        // 音量放大系数（根据 TTS 音频特性调整）
const SMOOTHING = 0.3           // 平滑因子（0=无平滑，1=完全不变）

export default function useLipSync(live2dRef) {
  const audioContextRef = useRef(null)
  const analyserRef = useRef(null)
  const sourceRef = useRef(null)
  const rafIdRef = useRef(null)
  const currentAudioRef = useRef(null)
  const prevValueRef = useRef(0)

  const stopLipSync = useCallback(() => {
    // 1. 取消 rAF
    if (rafIdRef.current) {
      cancelAnimationFrame(rafIdRef.current)
      rafIdRef.current = null
    }
    // 2. 断开音频节点
    if (sourceRef.current) {
      sourceRef.current.disconnect()
      sourceRef.current = null
    }
    // 3. 复位嘴巴参数
    if (live2dRef.current) {
      live2dRef.current.setMouthOpenY(0)
    }
    prevValueRef.current = 0
    // 4. 停止 Audio 元素
    if (currentAudioRef.current) {
      currentAudioRef.current.pause()
      currentAudioRef.current = null
    }
  }, [live2dRef])

  const playWithLipSync = useCallback((audioBase64) => {
    // 先停止上一次的口型同步
    stopLipSync()

    try {
      // 解码 base64 → Blob → Object URL
      const blob = new Blob(
        [Uint8Array.from(atob(audioBase64), c => c.charCodeAt(0))],
        { type: 'audio/mpeg' }
      )
      const url = URL.createObjectURL(blob)
      const audio = new Audio(url)
      currentAudioRef.current = audio

      // 惰性创建 AudioContext（浏览器要求用户交互后才能创建）
      if (!audioContextRef.current) {
        audioContextRef.current = new (window.AudioContext || window.webkitAudioContext)()
      }
      const ctx = audioContextRef.current

      // 创建 AnalyserNode
      const analyser = ctx.createAnalyser()
      analyser.fftSize = 256
      analyser.smoothingTimeConstant = 0.5
      analyserRef.current = analyser

      // Audio 元素 → MediaElementSource → AnalyserNode → destination
      const source = ctx.createMediaElementSource(audio)
      source.connect(analyser)
      analyser.connect(ctx.destination)
      sourceRef.current = source

      // rAF 循环：读取音量 → 映射嘴巴参数
      const dataArray = new Uint8Array(analyser.frequencyBinCount)

      const updateMouth = () => {
        analyser.getByteFrequencyData(dataArray)

        // 计算平均音量 (0~255 → 0~1)
        let sum = 0
        for (let i = 0; i < dataArray.length; i++) {
          sum += dataArray[i]
        }
        let volume = sum / dataArray.length / 255

        // 应用阈值和缩放
        if (volume < VOLUME_THRESHOLD) {
          volume = 0
        } else {
          volume = Math.min(1.0, (volume - VOLUME_THRESHOLD) * VOLUME_SCALE)
        }

        // 平滑处理（避免嘴巴抖动）
        const smoothed = prevValueRef.current * SMOOTHING + volume * (1 - SMOOTHING)
        prevValueRef.current = smoothed

        // 设置 Live2D 嘴巴参数
        if (live2dRef.current) {
          live2dRef.current.setMouthOpenY(smoothed)
        }

        rafIdRef.current = requestAnimationFrame(updateMouth)
      }

      // 音频开始播放后启动循环
      audio.onplay = () => {
        // 确保 AudioContext 已激活（应对 suspended 状态）
        if (ctx.state === 'suspended') ctx.resume()
        updateMouth()
      }

      // 音频结束时停止
      audio.onended = () => {
        stopLipSync()
        URL.revokeObjectURL(url)
      }

      audio.onerror = () => {
        stopLipSync()
        URL.revokeObjectURL(url)
      }

      audio.play().catch(e => {
        console.error('[LipSync] Audio play error:', e)
        stopLipSync()
      })
    } catch (e) {
      console.error('[LipSync] Setup error:', e)
      stopLipSync()
    }
  }, [live2dRef, stopLipSync])

  return { playWithLipSync, stopLipSync }
}
```

## 关键设计决策

| 问题 | 决策 |
|------|------|
| 音量分析方式 | `getByteFrequencyData`（频域），比 `getByteTimeDomainData` 更稳定 |
| 为什么不用 wlipsync 库 | 项目体量小，简单音量映射足够，不引入额外依赖 |
| AudioContext 复用 | 全局复用单个 AudioContext（惰性创建），避免浏览器限制 |
| 平滑处理 | 线性插值 `SMOOTHING=0.3`，防止嘴巴在安静段快速闪烁 |
| MediaElementSource 限制 | 同一 Audio 元素只能创建一个 MediaElementSource，因此每次新音频创建新 Audio 元素 |
| 音频播放中发新消息 | `playWithLipSync` 开头调用 `stopLipSync()` 停止上一段 |

## Spec 10: Streaming Expression (流式表情解析)

```
# 10-streaming-expression — 流式表情解析

**Status: 🔲 Pending**

## 背景与目标

当前表情处理流程：
1. LLM 流式输出完整 JSON 文本
2. 所有 text 事件发送完毕
3. 在 `chat_service.py` 中调用 `extract_expression()` 解析完整文本
4. 发送 expression 事件
5. 最后发送 audio 事件

问题：表情总是在文字全部显示完之后才出现，用户看到一长串文字后才看到角色变换表情，体验不佳。

**目标**：在 LLM 流式输出过程中，一旦检测到 `"expression"` 字段的值，立即发送 expression 事件，使表情与文字同步出现。

- `backend/main.py`
- `backend/prompts/reply.txt`
- `frontend/` 目录下的任何文件（前端已支持 onExpression 回调，无需改动）
- `backend/providers/`

## 依赖

- `06-backend-api`（SSE 流式架构）
- `07-frontend`（前端 onExpression 回调已就绪）

## 核心分析

### LLM 输出格式

LLM 被要求输出如下 JSON：
```json
{
  "reply": "哼，才不是呢！",
  "user_info": "用户喜欢猫",
  "expression": "脸红"
}
```

LLM 流式输出这个 JSON 时，token 是逐步生成的：
```
{"reply": "哼，才不 → 是呢！", "user_info": "用户 → 喜欢猫", "expression": " → 脸红"}
```

`expression` 字段通常在 JSON 末尾附近出现（按照 prompt 模板中的字段顺序）。但我们可以在流式输出过程中监测到 `"expression"` key 出现，并在其值完成后立即发送 expression 事件。

### 策略

使用**增量正则扫描**：在累积的 raw text 中寻找 `"expression"\s*:\s*"([^"]*)"` 模式。一旦匹配成功，立即发送 expression 事件。由于 expression 的值是一个短字符串（如 "脸红"），一般在几个 token 内就能完成。

---

## 实现要求

### `chat_service.py` 修改

将 `generate_reply_stream()` 方法改为在流式过程中检测 expression：

```python
import re
from typing import AsyncGenerator

class ChatService:
    # ...existing __init__...

    # 用于增量检测 expression 的正则
    _EXPRESSION_PATTERN = re.compile(r'"expression"\s*:\s*"([^"]*)"')

    async def generate_reply_stream(
        self, message: str
    ) -> AsyncGenerator[dict[str, str], None]:
        """流式生成回复，按事件类型 yield 字典"""
        chunks: list[str] = []
        expression_sent = False  # 标记：expression 事件是否已发送

        async for chunk in self.main_agent.reply_stream(message):
            if not chunk:
                continue
            chunks.append(chunk)
            yield {"type": "text", "content": chunk}

            # 增量检测 expression（仅在未发送过 expression 时检测）
            if not expression_sent:
                accumulated = "".join(chunks)
                match = self._EXPRESSION_PATTERN.search(accumulated)
                if match:
                    expression = match.group(1).strip()
                    if expression:
                        yield {"type": "expression", "content": expression}
                        expression_sent = True

        # 兜底：如果流式过程中未检测到 expression，从完整文本中提取
        if not expression_sent:
            full_text = "".join(chunks)
            expression = self.main_agent.extract_expression(full_text)
            if expression:
                yield {"type": "expression", "content": expression}

        # 合成语音（TTS 未实现时静默跳过）
        try:
            tts = get_tts()
            if tts.is_configured():
                full_text = "".join(chunks)
                clean_text = self.main_agent.extract_reply_text(full_text)
                audio_bytes = await tts.synthesize(clean_text)
                if audio_bytes:
                    audio_b64 = base64.b64encode(audio_bytes).decode("ascii")
                    yield {"type": "audio", "content": audio_b64}
        except NotImplementedError:
            pass
```

### 优化：避免每次 chunk 都做全文正则扫描

为了避免在每个 text chunk 到达时都对已累积的全文做正则匹配（O(n^2) 复杂度），使用一个**扫描偏移量**，只对新增部分及其前缀窗口做检测：

```python
async def generate_reply_stream(
    self, message: str
) -> AsyncGenerator[dict[str, str], None]:
    chunks: list[str] = []
    expression_sent = False
    scan_offset = 0  # 上次扫描到的位置

    async for chunk in self.main_agent.reply_stream(message):
        if not chunk:
            continue
        chunks.append(chunk)
        yield {"type": "text", "content": chunk}

        if not expression_sent:
            accumulated = "".join(chunks)
            # 从 scan_offset 前的安全窗口开始扫描（"expression" 关键字可能跨 chunk）
            search_start = max(0, scan_offset - 30)
            match = self._EXPRESSION_PATTERN.search(accumulated, search_start)
            if match:
                expression = match.group(1).strip()
                if expression:
                    yield {"type": "expression", "content": expression}
                    expression_sent = True
            scan_offset = len(accumulated)

    # 兜底
    if not expression_sent:
        full_text = "".join(chunks)
        expression = self.main_agent.extract_expression(full_text)
        if expression:
            yield {"type": "expression", "content": expression}

    # TTS（unchanged）
    try:
        tts = get_tts()
        if tts.is_configured():
            full_text = "".join(chunks)
            clean_text = self.main_agent.extract_reply_text(full_text)
            audio_bytes = await tts.synthesize(clean_text)
            if audio_bytes:
                audio_b64 = base64.b64encode(audio_bytes).decode("ascii")
                yield {"type": "audio", "content": audio_b64}
    except NotImplementedError:
        pass
```

## SSE 事件顺序变化

### 之前（Spec 06 定义）：
```
text → text → text → ... → expression → audio → done
```

### 之后（本 spec）：
```
text → text → expression → text → ... → audio → done
                ↑
         expression 可能出现在 text 流中间
```

**前端兼容性**：前端 `client.js` 的 SSE 解析是按 `type` 字段 switch-case 分发的，不依赖事件顺序。`onExpression` 回调在任何时候被调用都能正确设置 Live2D 表情。因此前端无需任何修改。

## 关键设计决策

| 问题 | 决策 |
|------|------|
| 是否修改 reply.txt 调整字段顺序 | 否。CONVENTIONS.md 禁止修改 reply.txt。通过增量扫描解决 |
| 正则 vs JSON 流式解析器 | 正则。expression 值是简单字符串，不含嵌套结构，正则足够 |
| expression 出现在流的哪个位置 | 取决于 LLM 输出顺序。通常 reply 先输出，expression 在后半段 |
| expression 检测失败 | 兜底到 extract_expression()（与当前行为一致） |
| 性能考虑 | scan_offset 窗口扫描避免 O(n^2)，对短回复（15字内）无性能问题 |
| expression 事件是否可能重复 | 不会。expression_sent 布尔标记确保只发送一次 |

# 11-vector-memory — 向量化记忆检索

**Status: 🔲 Pending**

## 背景与目标

当前 `conversation.py` 的 `retrieve()` 使用简单的关键词匹配和字符重叠评分来检索记忆摘要。这种方式存在严重局限：
1. 无法理解语义相似性（"我讨厌吃辣" 和 "不喜欢辣椒" 无法匹配）
2. 单字符重叠评分噪声大，精度低
3. 中文分词缺失，检索质量差

**目标**：引入 Embedding 向量检索，替代关键词匹配，使记忆检索具备语义理解能力。

**设计原则**（借鉴 OpenClaw/AIRI）：
- 保持简单：不引入 ChromaDB 或外部向量库，使用纯 Python + JSON 存储
- 多级降级：Embedding API 失败 → 关键词匹配兜底
- 缓存友好：按内容 hash 缓存 embedding，避免重复计算
- 配置化：Embedding provider 通过 config.json 配置

        self.provider_name = provider_name
        self.api_key = config.get("api_key", "")
        self.model = config.get("model", "text-embedding-3-small")
        self.base_url = config.get("base_url", "https://api.openai.com/v1")

    async def embed(self, text: str) -> list[float]:
        """调用 /embeddings 端点获取单条文本的 embedding"""
        async with httpx.AsyncClient(
            timeout=httpx.Timeout(connect=10, read=30, write=10, pool=10),
            verify=False
        ) as client:
            response = await client.post(
                f"{self.base_url.rstrip('/')}/embeddings",
                headers={
                    "Authorization": f"Bearer {self.api_key}",
                    "Content-Type": "application/json",
                },
                json={
                    "input": text,
                    "model": self.model,
                },
            )
            response.raise_for_status()
            data = response.json()
            return data["data"][0]["embedding"]

    async def embed_batch(self, texts: list[str]) -> list[list[float]]:
        """批量获取 embedding（OpenAI API 原生支持 batch input）"""
        if not texts:
            return []
        async with httpx.AsyncClient(
            timeout=httpx.Timeout(connect=10, read=60, write=10, pool=10),
            verify=False
        ) as client:
            response = await client.post(
                f"{self.base_url.rstrip('/')}/embeddings",
                headers={
                    "Authorization": f"Bearer {self.api_key}",
                    "Content-Type": "application/json",
                },
                json={
                    "input": texts,
                    "model": self.model,
                },
            )
            response.raise_for_status()
            data = response.json()
            # 按 index 排序确保顺序正确
            sorted_data = sorted(data["data"], key=lambda x: x["index"])
            return [item["embedding"] for item in sorted_data]

    def is_configured(self) -> bool:
        if self.provider_name == "ollama":
            return True
        return bool(self.api_key)

### embedding_utils.py

```python
# backend/embedding_utils.py

import hashlib
import json
import math
import os
from typing import Optional


def cosine_similarity(a: list[float], b: list[float]) -> float:
    """计算两个向量的余弦相似度"""
    if len(a) != len(b):
        return 0.0
    dot = sum(x * y for x, y in zip(a, b))
    norm_a = math.sqrt(sum(x * x for x in a))
    norm_b = math.sqrt(sum(x * x for x in b))
    if norm_a == 0 or norm_b == 0:
        return 0.0
    return dot / (norm_a * norm_b)


def l2_normalize(vec: list[float]) -> list[float]:
    """L2 归一化向量"""
    norm = math.sqrt(sum(x * x for x in vec))
    if norm == 0:
        return vec
    return [x / norm for x in vec]


def content_hash(text: str) -> str:
    """计算文本内容的 SHA-256 hash（用于缓存键）"""
    return hashlib.sha256(text.encode("utf-8")).hexdigest()[:16]


class EmbeddingCache:
    """基于 JSON 文件的 embedding 缓存"""

    def __init__(self, cache_path: str, max_entries: int = 500):
        self.cache_path = cache_path
        self.max_entries = max_entries
        self.cache: dict[str, list[float]] = self._load()

    def _load(self) -> dict[str, list[float]]:
        try:
            if os.path.exists(self.cache_path):
                with open(self.cache_path, "r", encoding="utf-8") as f:
                    return json.load(f)
        except Exception as e:
            print(f"[EmbeddingCache] 加载缓存失败: {e}")
        return {}

    def _save(self) -> None:
        os.makedirs(os.path.dirname(self.cache_path), exist_ok=True)
        with open(self.cache_path, "w", encoding="utf-8") as f:
            json.dump(self.cache, f, ensure_ascii=False)

    def get(self, text: str) -> Optional[list[float]]:
        """根据文本 hash 查找缓存的 embedding"""
        key = content_hash(text)
        return self.cache.get(key)

    def put(self, text: str, embedding: list[float]) -> None:
        """缓存 embedding，超过上限时淘汰最早的条目"""
        key = content_hash(text)
        self.cache[key] = embedding
        # 简单淘汰：超过上限时删除前 10% 的条目
        if len(self.cache) > self.max_entries:
            keys = list(self.cache.keys())
            remove_count = len(keys) // 10
            for k in keys[:remove_count]:
                del self.cache[k]
        self._save()
```

### `_get_embedding()` 方法

```python
async def _get_embedding(self, text: str) -> list[float] | None:
    """获取文本的 embedding 向量，优先使用缓存"""
    if not self.embedding_provider:
        return None

    # 检查缓存
    cached = self.embedding_cache.get(text)
    if cached is not None:
        return cached

    try:
        from embedding_utils import l2_normalize
        embedding = await self.embedding_provider.embed(text)
        embedding = l2_normalize(embedding)
        self.embedding_cache.put(text, embedding)
        return embedding
    except Exception as e:
        print(f"[Memory] Embedding 生成失败: {e}")
        return None
```

### `retrieve()` 改造

```python
def retrieve(self, user_message: str, n_results: int = 3) -> list[str]:
    """获取与用户消息最相关的历史记忆（向量检索 + 关键词兜底）"""
    if not self.summaries:
        return []

    # 尝试向量检索
    results = self._retrieve_vector(user_message, n_results)
    if results:
        return results

    # 降级：关键词匹配（与原逻辑一致）
    return self._retrieve_keyword(user_message, n_results)

async def retrieve_async(self, user_message: str, n_results: int = 3) -> list[str]:
    """异步版本的 retrieve（支持实时生成查询 embedding）"""
    if not self.summaries:
        return []

    # 尝试向量检索
    query_embedding = await self._get_embedding(user_message)
    if query_embedding:
        results = self._retrieve_by_embedding(query_embedding, n_results)
        if results:
            return results

    # 降级：关键词匹配
    return self._retrieve_keyword(user_message, n_results)

def _retrieve_vector(self, user_message: str, n_results: int) -> list[str]:
    """同步向量检索（使用缓存的查询 embedding）"""
    cached_embedding = self.embedding_cache.get(user_message)
    if not cached_embedding:
        return []  # 无缓存，降级到关键词
    return self._retrieve_by_embedding(cached_embedding, n_results)

def _retrieve_by_embedding(
    self, query_embedding: list[float], n_results: int
) -> list[str]:
    """根据查询向量在摘要中做余弦相似度排序"""
    from embedding_utils import cosine_similarity

    scored: list[tuple[float, str]] = []
    for item in self.summaries:
        emb = item.get("embedding")
        if not emb:
            continue
        score = cosine_similarity(query_embedding, emb)
        if score > 0.3:  # 最低相似度阈值
            scored.append((score, item["summary"]))

    scored.sort(key=lambda x: x[0], reverse=True)
    return [s[1] for s in scored[:n_results]]

def _retrieve_keyword(self, user_message: str, n_results: int) -> list[str]:
    """关键词匹配检索（原 retrieve 逻辑）"""
    scored = []
    for item in self.summaries:
        score = 0
        topics = [t.strip() for t in item.get("topics", "").split(",") if t.strip()]
        for topic in topics:
            if topic in user_message:
                score += 2
        summary = item.get("summary", "")
        for char in set(user_message):
            if char in summary and char.strip():
                score += 0.1
        if score > 0:
            scored.append((score, item["summary"]))

    scored.sort(key=lambda x: x[0], reverse=True)
    results = [s[1] for s in scored[:n_results]]

    if not results:
        results = [item["summary"] for item in self.summaries[-n_results:]]
    return results
```

## 关键设计决策

| 问题 | 决策 |
|------|------|
| 向量库选择 | 纯 JSON + Python cosine，不引入 ChromaDB/FAISS（摘要量级 <1000 条足够） |
| Embedding 缓存 | JSON 文件，按内容 hash 缓存，最多 500 条 |
| 检索降级 | 向量检索无结果 → 关键词匹配 → 最近 N 条兜底 |
| Embedding 为 None | 旧数据无 embedding 的条目在向量检索中被跳过，进入关键词兜底 |
| 相似度阈值 | 0.3（cosine similarity），低于此分数视为不相关 |
| retrieve 保持同步签名 | 新增 retrieve_async 异步版本，原 retrieve() 签名保持不变（供 tools.py 调用） |
| Embedding 维度 | 不做硬编码，跟随 provider 返回的维度 |

- 不引入 ChromaDB、FAISS 或任何外部向量库依赖
- `retrieve()` 的签名不变（同步方法，关键词匹配）

# backend/database.py

import os
import sqlite3
from datetime import datetime
from typing import Optional

DB_PATH = os.path.join(os.path.dirname(__file__), "..", "save", "nana.db")


class Database:
    """SQLite 数据库管理，单例模式"""

    _instance: Optional["Database"] = None

    def __init__(self) -> None:
        os.makedirs(os.path.dirname(DB_PATH), exist_ok=True)
        self.conn = sqlite3.connect(DB_PATH)
        self.conn.row_factory = sqlite3.Row
        self.conn.execute("PRAGMA journal_mode=WAL")
        self._init_tables()

    @classmethod
    def get(cls) -> "Database":
        if cls._instance is None:
            cls._instance = Database()
        return cls._instance

    def _init_tables(self) -> None:
        self.conn.executescript("""
            CREATE TABLE IF NOT EXISTS turns (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                session_id TEXT NOT NULL DEFAULT 'default',
                user_message TEXT NOT NULL,
                assistant_message TEXT NOT NULL,
                created_at TEXT NOT NULL DEFAULT (datetime('now', 'localtime')),
                archived INTEGER NOT NULL DEFAULT 0
            );
            CREATE INDEX IF NOT EXISTS idx_turns_session
                ON turns(session_id, archived);
            CREATE INDEX IF NOT EXISTS idx_turns_created
                ON turns(session_id, created_at);

            CREATE TABLE IF NOT EXISTS sessions (
                session_id TEXT PRIMARY KEY,
                created_at TEXT NOT NULL DEFAULT (datetime('now', 'localtime')),
                last_active TEXT NOT NULL DEFAULT (datetime('now', 'localtime')),
                title TEXT DEFAULT ''
            );
        """)
        self.conn.commit()

    def save_turn(
        self, session_id: str, user_message: str, assistant_message: str
    ) -> int:
        """保存一轮对话，返回行 ID"""
        cursor = self.conn.execute(
            "INSERT INTO turns (session_id, user_message, assistant_message) VALUES (?, ?, ?)",
            (session_id, user_message, assistant_message),
        )
        self.conn.commit()
        # 更新会话最后活跃时间
        self.conn.execute(
            """INSERT INTO sessions (session_id, last_active)
               VALUES (?, datetime('now', 'localtime'))
               ON CONFLICT(session_id)
               DO UPDATE SET last_active = datetime('now', 'localtime')""",
            (session_id,),
        )
        self.conn.commit()
        return cursor.lastrowid

    def load_turns(
        self, session_id: str, limit: int = 20
    ) -> list[dict]:
        """加载指定会话的未归档对话（最近 limit 条）"""
        rows = self.conn.execute(
            """SELECT user_message, assistant_message, created_at
               FROM turns
               WHERE session_id = ? AND archived = 0
               ORDER BY id DESC
               LIMIT ?""",
            (session_id, limit),
        ).fetchall()
        # 反转为时间正序
        return [dict(r) for r in reversed(rows)]

    def mark_archived(self, session_id: str, count: int) -> None:
        """将最早的 count 条未归档记录标记为已归档"""
        self.conn.execute(
            """UPDATE turns SET archived = 1
               WHERE id IN (
                   SELECT id FROM turns
                   WHERE session_id = ? AND archived = 0
                   ORDER BY id ASC
                   LIMIT ?
               )""",
            (session_id, count),
        )
        self.conn.commit()

    def get_recent_messages(
        self, session_id: str, limit: int = 50
    ) -> list[dict]:
        """获取最近的消息（含已归档），用于前端恢复聊天界面"""
        rows = self.conn.execute(
            """SELECT user_message, assistant_message, created_at
               FROM turns
               WHERE session_id = ?
               ORDER BY id DESC
               LIMIT ?""",
            (session_id, limit),
        ).fetchall()
        return [dict(r) for r in reversed(rows)]

    def close(self) -> None:
        self.conn.close()
        Database._instance = None

| 问题 | 决策 |
|------|------|
| 存储选择 | SQLite（Python 内置，无额外依赖，事务支持） |
| WAL 模式 | 开启 `PRAGMA journal_mode=WAL`（提高并发读写性能） |
| 单例 vs 每次创建 | Database 单例，避免重复打开/关闭连接 |
| session 管理 | ChatService 内部缓存，按 session_id 创建独立 ConversationHistory |

| 线程安全 | SQLite 在 WAL 模式下支持单写多读，FastAPI 是单线程 async 不需要额外锁 |

## Spec 13: Message Queue & Concurrency Control (消息队列与并发控制)

```
# 13-concurrency-control — 消息队列与并发控制

**Status: 🔲 Pending**

## 背景与目标

当前系统在以下场景存在并发问题：

1. **前端快速连发**：用户快速按 Enter 发送多条消息，虽然前端有 `loading` 状态禁用输入，但极端情况下（如按 Enter 的同时按下语音识别）可能触发多个并发请求
2. **后端无锁**：多个 `/api/chat` 请求同时到达时，可能并发修改同一个 `ConversationHistory`（操作 `self.turns`、写 SQLite、写 JSON 文件），导致数据竞争
3. **旧请求滞留**：用户发送第二条消息时，第一条的 SSE 流仍在推送中，前端可能交替收到两个流的事件，导致消息混乱

**目标**：
1. 后端：每个 session 同一时刻只处理一个请求（session-level lock）
2. 前端：增强输入防抖，确保一次只有一个活跃请求
3. Generation ID：为每次对话生成唯一 ID，前端可丢弃旧的流

## 实现要求

### 1. 后端 Session Lock

#### chat_service.py

```python
import asyncio
import uuid
from typing import AsyncGenerator

class ChatService:
    def __init__(self) -> None:
        self.llm_adapter = LLMProviderAdapter()
        self._sessions: dict[str, tuple] = {}
        self._locks: dict[str, asyncio.Lock] = {}  # 新增：session-level locks
        self._active_generation: dict[str, str] = {}  # 新增：session → 当前 generation_id

    def _get_lock(self, session_id: str) -> asyncio.Lock:
        """获取或创建指定 session 的锁"""
        if session_id not in self._locks:
            self._locks[session_id] = asyncio.Lock()
        return self._locks[session_id]

    async def generate_reply_stream(
        self, message: str, session_id: str = "default"
    ) -> AsyncGenerator[dict[str, str], None]:
        """带 session 锁的流式生成"""
        generation_id = str(uuid.uuid4())[:8]
        lock = self._get_lock(session_id)

        # 标记当前 generation
        self._active_generation[session_id] = generation_id

        # 发送 generation_id 作为首个事件
        yield {"type": "generation_id", "content": generation_id}

        async with lock:
            # 获取锁后检查：如果已被新请求取代，直接结束
            if self._active_generation.get(session_id) != generation_id:
                print(f"[ChatService] generation {generation_id} 已被取代，跳过")
                return

            conversation, agent = self._get_session(session_id)

            chunks: list[str] = []
            expression_sent = False
            scan_offset = 0

            async for chunk in agent.reply_stream(message):
                # 每个 chunk 前检查是否仍是活跃 generation
                if self._active_generation.get(session_id) != generation_id:
                    print(f"[ChatService] generation {generation_id} 中途被取代")
                    return

                if not chunk:
                    continue
                chunks.append(chunk)
                yield {"type": "text", "content": chunk}

                # expression 检测 (from spec 10)
                if not expression_sent:
                    accumulated = "".join(chunks)
                    search_start = max(0, scan_offset - 30)
                    match = self._EXPRESSION_PATTERN.search(accumulated, search_start)
                    if match:
                        expression = match.group(1).strip()
                        if expression:
                            yield {"type": "expression", "content": expression}
                            expression_sent = True
                    scan_offset = len(accumulated)

            # 兜底 expression
            if not expression_sent:
                full_text = "".join(chunks)
                expression = agent.extract_expression(full_text)
                if expression:
                    yield {"type": "expression", "content": expression}

            # TTS
            try:
                tts = get_tts()
                if tts.is_configured():
                    full_text = "".join(chunks)
                    clean_text = agent.extract_reply_text(full_text)
                    audio_bytes = await tts.synthesize(clean_text)
                    if audio_bytes:
                        audio_b64 = base64.b64encode(audio_bytes).decode("ascii")
                        yield {"type": "audio", "content": audio_b64}
            except NotImplementedError:
                pass
```

function App() {
  // ...existing state...
  const activeGenerationRef = useRef(null)  // 当前活跃的 generation ID
  const abortControllerRef = useRef(null)   // 用于中断旧请求

  const handleSendMessage = (directMessage) => {
    const text = directMessage ?? input
    if (!text.trim() || loading) return
    if (!directMessage) setInput('')
    setLoading(true)

    // 中断上一个未完成的请求
    if (abortControllerRef.current) {
      abortControllerRef.current.abort()
    }
    const abortController = new AbortController()
    abortControllerRef.current = abortController

    setMessages(prev => [...prev, { type: 'user', content: text }])

    let rawAccumulator = ''
    let myGenerationId = null

    api.chatStream(text, {
      signal: abortController.signal,  // 传入 abort signal

      onGenerationId: (id) => {
        myGenerationId = id
        activeGenerationRef.current = id
      },
      onText: (chunk) => {
        // 检查是否仍是活跃 generation
        if (myGenerationId && activeGenerationRef.current !== myGenerationId) {
          // 旧流，中断
          abortController.abort()
          return
        }
        rawAccumulator += chunk
        const replyText = extractReply(rawAccumulator)
        if (replyText !== null) {
          setMessages(prev => {
            const last = prev[prev.length - 1]
            if (last?.type === 'assistant') {
              return [...prev.slice(0, -1), { type: 'assistant', content: replyText }]
            }
            return [...prev, { type: 'assistant', content: replyText }]
          })
        }
      },
      onExpression: (expression) => {
        if (myGenerationId && activeGenerationRef.current !== myGenerationId) return
        if (live2dRef.current) live2dRef.current.showExpression(expression)
      },
      onAudio: (audioBase64) => {
        if (myGenerationId && activeGenerationRef.current !== myGenerationId) return
        playWithLipSync(audioBase64)  // from spec 09
      },
      onDone: () => {
        if (myGenerationId && activeGenerationRef.current !== myGenerationId) return
        if (rawAccumulator) {
          setMessages(prev => {
            const last = prev[prev.length - 1]
            if (last?.type !== 'assistant') {
              const fallback = extractReply(rawAccumulator) ?? rawAccumulator
              return [...prev, { type: 'assistant', content: fallback }]
            }
            return prev
          })
        }
        setLoading(false)
      },
      onError: (msg) => {
        if (myGenerationId && activeGenerationRef.current !== myGenerationId) return
        console.error('Chat error:', msg)
        setLoading(false)
      },
    })
  }

  // ...rest unchanged...
}

#### api/client.js

支持 AbortController 和 `generation_id` 事件：

```javascript
export const api = {
  chatStream(message, callbacks = {}) {
    const {
      onText, onExpression, onAudio, onDone, onError,
      onGenerationId,  // 新增
      signal,          // 新增：AbortSignal
    } = callbacks

    const run = async () => {
      const response = await fetch(`${BASE_URL}/api/chat`, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ message, session_id: "default" }),
        signal,  // 传入 AbortSignal
      })

      if (!response.ok) {
        throw new Error(await readError(response))
      }

      if (!response.body) {
        throw new Error("Empty response body")
      }

      const reader = response.body.getReader()
      const decoder = new TextDecoder()
      let buffer = ""

      while (true) {
        const { done, value } = await reader.read()
        if (done) break

        buffer += decoder.decode(value, { stream: true })

        let boundary = buffer.indexOf("\n\n")
        while (boundary !== -1) {
          const event = buffer.slice(0, boundary)
          buffer = buffer.slice(boundary + 2)

          for (const line of event.split("\n")) {
            if (!line.startsWith("data:")) continue
            const data = line.slice(5).trim()
            try {
              const parsed = JSON.parse(data)
              switch (parsed.type) {
                case "generation_id":
                  onGenerationId?.(parsed.content)
                  break
                case "text":
                  onText?.(parsed.content)
                  break
                case "expression":
                  onExpression?.(parsed.content)
                  break
                case "audio":
                  onAudio?.(parsed.content)
                  break
                case "done":
                  onDone?.()
                  return
                case "error":
                  onError?.(parsed.content)
                  return
              }
            } catch {
              // 忽略解析失败的行
            }
          }

          boundary = buffer.indexOf("\n\n")
        }
      }
    }

    run().catch((err) => {
      // AbortError 是正常中断，不报错
      if (err.name === 'AbortError') return
      onError?.(err.message)
    })
  },

  // ...other methods unchanged...
}

## 并发场景处理

### 场景 1：用户快速连发 2 条消息

```
T=0: 用户发 "你好"
  → 前端创建 AbortController A, generationId = g1
  → 后端收到请求，设 active_generation["default"] = g1
  → 后端获取 session lock，开始处理

T=0.5: 用户发 "今天天气如何"（input 虽 disabled，但假设通过语音输入）
  → 前端调用 abortControllerRef.current.abort()，中断 g1 的 fetch
  → 前端创建 AbortController B, generationId = g2
  → 后端收到请求，设 active_generation["default"] = g2
  → 后端尝试获取 session lock → 等待 g1 完成

  g1 的流在下个 chunk 检查到 active_generation != g1 → return
  lock 释放 → g2 获取 lock → 开始处理

结果：g1 被中断，g2 正常完成
```

### 场景 2：后端正在处理时前端页面刷新

```
后端 g1 正在流式输出 → 前端断开 SSE 连接
后端的 StreamingResponse 会检测到 client disconnect → 停止 yield
lock 自然释放
```

### 场景 3：LLM 响应非常慢，用户等不及发新消息

```
g1 正在等待 LLM 响应 → 前端 abort g1 → 后端 g1 在下个 yield 检查到被取代 → return
g2 等待 lock → 获取 lock → 正常处理
```

## 关键设计决策

| 问题 | 决策 |
|------|------|
| 锁粒度 | Session 级别（每个 session 一把锁） |
| 锁类型 | `asyncio.Lock`（适用于 FastAPI 的 async 模型） |
| 旧请求处理 | 后端检测 generation_id 不匹配后 return，前端 abort fetch |
| Generation ID 格式 | UUID 前 8 位（足够区分快速连发的请求） |
| 前端防抖 | loading 状态 + AbortController 双重保护 |
| generation_id 事件 | 在获取锁之前发送，保证前端立即收到 |

---

## 约束

- `asyncio.Lock` 是可重入安全的（FastAPI 在同一个 event loop 上执行所有请求）
- `generation_id` 事件对前端是可选的（旧前端忽略未知 type 即可）
- 不修改 `main_agent.py` 和 `conversation.py`
- AbortError 在前端静默处理，不显示错误
- 锁等待不设超时（依赖 LLM 自身的超时机制）
- generation_id 检查是轻量操作（字典查找），不影响流式性能
